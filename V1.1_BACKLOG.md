## v1.1

Priority order within v1.1:

1. **U4 — Review-before-harvest preview on mark_sent (highest priority).**
   The voice corpus is a compounding asset: every sent issue permanently shapes
   future generation, and there's currently no undo and no per-example exclude
   at harvest time (cleanup is manual on the Voice Examples page). Add a preview
   step on "Mark as Sent" that shows the 5 sections about to be harvested and
   lets Neelabh deselect any before they enter the corpus.

2. **U3 — Un-send / delete-issue endpoint.** No `DELETE /newsletter/issues/{id}`
   exists; a mistaken or test issue is permanent in the archive (had to clear
   the Phase-7 dry-run issue via direct DB + filesystem deletion). Add a delete
   (and optionally a "demote to draft") endpoint + archive UI affordance.

3. **U2 — Auto-discard untouched drafts after 7 days.** "+ New Draft" creates a
   draft immediately on click, so abandoned empty drafts accumulate (13 found
   during the dry run). Sweep drafts with no content older than 7 days.

4. **U5 — Issue-number preview before send.** Composer shows "Unsent" with no
   indication of which number the issue will take. Show "This will be Issue 0NN"
   (next sequential) in the composer header / send modal.

5. **U6 + U7 — Voice-check tuning pass (bundle).**
   - U6: soft caps not enforced — generation can exceed limits (a What's Moving
     line came back 26 words vs the 25-word guidance), and voice rule 7 only
     flags >30-word sentences, so 25–30w lines slip through.
   - U7: voice check over-flags legitimate quotable predictions (flagged "By
     Q4 2026, half the agentic pilots launched this year will be quietly
     shelved…" on rule 7). Tune the rubric so earned long sentences pass.

6. **U8 — Exercise PDF spotlight image.** All 6 seeded POVs have
   `demo_screenshot_path=null`, so the inline-image path in `build_pdf` has
   never run with real data. Verify rendering when the first POV screenshot is
   uploaded.
